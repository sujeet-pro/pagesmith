# Choose Your Path

Pagesmith offers two packages for different needs. Pick the one that matches your goal.

## I want a documentation site

Use **@pagesmith/docs** -- a convention-based docs site builder with zero custom code needed.

**You get:** Navigation from folder structure, Pagefind search, dark mode, responsive layout, and a polished default theme -- all from a single config file.

**Best for:** API docs, project documentation, knowledge bases, guides.

-> [Get started with @pagesmith/docs](/guide/docs-getting-started/)

## I want a custom site with structured content

Use **@pagesmith/core** -- a content layer with schema-validated collections, a rich markdown pipeline, and Vite plugins.

**You get:** Full control over layout, styling, and rendering. Use any framework (React, Solid, Svelte) or template engine (EJS, Handlebars).

**Best for:** Blogs, portfolios, marketing sites, custom documentation, any site where you want structured content with your own design.

-> [Get started with @pagesmith/core](/guide/getting-started/)

## Not sure?

| Question | @pagesmith/docs | @pagesmith/core |
|----------|----------------|-----------------|
| Do I need custom layouts? | Optional overrides | Full control |
| Do I need a specific framework? | Not needed | React, Solid, Svelte, etc. |
| How fast to get started? | Minutes | ~30 minutes |
| Search built in? | Yes (Pagefind) | Add it yourself |
| Navigation from folders? | Automatic | Build your own |

**Start with `@pagesmith/docs`** if you're unsure -- you can always drop down to `@pagesmith/core` later for more control.
